MySQL

Une base de données est une collection de données organisée de façon à être facilement accessible, administrée et mise à jour. Les bases de données peuvent être classées par le type de contenu qu’elles renferment : bibliographique, full text, images ou numériques.

Le terme Base de Données est apparu en 1964, s'en suivirent de nombreux modèles de bases de données expérimentales et factuelles, d'abord utilisées par l'armée et les spécialistes du sujet. Apparues dans les années 1990, les bases de données objet-relationnel permettent d'organiser les données sous la forme de tables définies de façon formelle, à partir desquelles les données sont accessibles et assemblées sans avoir à réorganiser les tables de la base de données.

Nous utiliserons lors de nos cours le système de gestion de bases de données relationnelles (SGBDR) MySQL.

MySQL fait partie des logiciels de gestion de base de données les plus utilisés au monde, (applications web principalement) par les professionnels.

Depuis mai 2009, son créateur Michael Widenius a créé MariaDB pour continuer son développement en tant que projet Open Source.

Pour nous familiariser avec ce système, rien de mieux que la pratique !


Installation en interface graphique.

Pour être le plus utilisé et libre, nous allons commencer par ceci :

Les commandes pour l'installation et le téléchargement peuvent être trouvées sur le site https://www.mysql.com.

Installation depuis le terminal.

Entrer les commandes suivantes dans un terminal :

Installer les paquets mysql client

sudo apt-get install mysql-client

Installer les paquets mysql server

sudo apt-get install mysql-server

phpMyAdmin (PMA) est une application Web de gestion pour les systèmes de gestion de base de données MySQL réalisée en PHP et distribuée sous licence GNU GPL.

Installer les paquets de phpmyadmin

sudo apt-get install phpmyadmin

Une fois ces commandes exécutées, tous les paquets nécessaires sont maintenant installés et vous pouvez commencer à interagir avec la BDD.

Toujours depuis le terminal, vous pouvez vous connecter à mysql via la commande suivante :

mysql -u root -p

Il vous sera demandé un mot de passe, celui que vous aurez saisi lors de l'installation des paquets précédents.

Entrez le sur votre clavier et appuyez sur Entrée.Vous êtes maintenant connectés à mysql et pouvez interagir avec vos bases de données, voici donc quelques commandes.

Commandes relatives aux Bases de Données

Créer une Base de Données

CREATE DATABASE [nom_de_la_base_de_données];

Supprimer une Base de Données

DROP DATABASE [nom_de_la_base_de_données];

Lister les Bases de Données

SHOW DATABASES;

Spécifier la Base de Données avec laquelle interagir

USE [nom_de_la_base_de_données];

Commandes relatives aux Tables

Créer une Table dans une Base de Données

CREATE TABLE [nom_de_la_table];

Afficher la structure d'une table

DESCRIBE [nom_de_la_table];

Supprimer une Table

DROP TABLE [nom_de_la_table];

Renommer une Table

ALTER TABLE [nom_de_la_table] RENAME AS [nouveau_nom_de_la_table];

Vider une Table

TRUNCATE [nom_de_la_table];

Les types de données

Les types de données utilisées sont définis lors de la création de votre table.

Voici quelques types de données que vous pourriez rencontrer :

Le type INT : Nombres entiers, l'id est habituellement de ce type.

Le type DECIMAL : Nombres décimaux.

Le type CHAR/VARCHAR : Permet de saisir du texte et des chaines de caractères, avec une taille fixe maximale de 255 caractères.

Le type DATE : Permet de saisir des dates, sous le format AAAA-MM-JJ.

Exemple de création de table :

CREATE TABLE [nom_de_la_table] (

id INT NOT NULL PRIMARY KEY AUTO_INCREMENT,,

nom VARCHAR(20),,

prenom VARCHAR(20),,

phone VARCHAR(20),

datedenaissance DATE);

Un exemple de selection sur une table donnée.

SELECT * FROM nom_de_la_table; : Affiche tout le contenu de la table.

DESCRIBE nom_de_la_base_de_donnees; : Affiche le contenu de la base de données.

La syntaxe d’UNION est simplissime :il vous suffit d’ajouter UNION entre deux requêtes pour les unir. Le nombre de requêtes qu'il est possible d'unir successivement est illimité.

SELECT... UNION SELECT...

Exemple d'union :

SELECT Animal.* FROM Animal

INNER JOIN Espece ON Animal.espece_id = Espece.id

WHERE Espece.nom_courant = 'Chat'

UNION

SELECT Animal.* FROM Animal

INNER JOIN Espece ON Animal.espece_id = Espece.id

WHERE Espece.nom_courant = 'Tortue d''Hermann';
